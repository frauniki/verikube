## Features

<CardGrid stagger>
	<Card title="Declarative & GitOps-friendly" icon="seti:git">
		Checks are CRDs: suites live in Git, results live in `.status`. No
		external state, no separate database.
	</Card>
	<Card title="Run from where it matters" icon="setting">
		Place runner pods with `nodeSelector`, `tolerations` and
		`topologySpread` — verify reachability from the exact nodes your
		workloads run on.
	</Card>
	<Card title="Three probe types" icon="puzzle">
		TCP connect, HTTP (status codes and headers), and the standard gRPC
		Health Checking Protocol.
	</Card>
	<Card title="Negative tests" icon="warning">
		Assert that a connection **fails** with `expect: Failure` — verify
		that security groups and network policies actually block traffic.
	</Card>
	<Card title="Cron scheduling + run-now" icon="pencil">
		Standard 5-field cron schedules, or trigger a run ad hoc with a
		single annotation.
	</Card>
	<Card title="Prometheus metrics" icon="magnifier">
		Live pass/fail state per check, probe latency histograms and run
		counters — Grafana-ready out of the box.
	</Card>
</CardGrid>
